# Dolphin body-site microbiome — analysis code

Developed and maintained by Shuai Gu, shuai.gu@yahoo.com
Code to reproduce the bioinformatic and statistical analyses and the figures in
Fiori et al., *Sex, contraceptives, and social grouping shape the structure of
dolphin-associated microbiomes* (submitted to *Animal Microbiome*).

The analysis characterizes 16S rRNA (V3–V4) amplicon microbiomes across six body
sites in common bottlenose dolphins (*Tursiops truncatus*) and rough-toothed dolphins
(*Steno bredanensis*) housed across six facilities, using QIIME 2 for read processing
and taxonomy and a compositional (CLR / Aitchison) approach for beta diversity.

## Repository layout

```
.
├── 01_read_processing/   per-tube demux/trim/denoise + merge + classify
├── 02_classifier/        SILVA 138.2 V3–V4 classifier build (see its own README)
├── 03_filtering/         mitochondria/chloroplast removal
├── 04_taxonomic/         phylum/genus collapse + Figures 1, 2, 5
├── 05_alpha/             alpha diversity + Figure 3 + alpha stats
├── 06_beta/              CLR/Aitchison PCoA panels (Figure 4)
├── metadata/             merged_dolphin_4.txt
└── palettes/             palettes_pub.py
```

## Inputs you need (not included here)

- **Raw sequencing reads** — deposited at NCBI SRA under BioProject **PRJNA1492278**.
- **SILVA 138.2 reference database** — downloaded from [arb-silva.de](https://www.arb-silva.de/);
  see `02_classifier/README.md`.
- **Sample metadata** — `metadata/merged_dolphin_4.txt` (included in this repo).

Large intermediate artifacts (`.qza` / `.qzv` feature tables, distance matrices,
ordinations, the trained classifier) are **not** committed; they are regenerated by
running the pipeline below.

## Environment

All QIIME 2 steps and all custom analysis/plotting scripts were run under
**QIIME 2 2024.2** (`module load qiime2/2024.2`), which provides the Python stack used
by the custom scripts: NumPy, pandas, SciPy, scikit-bio, and Matplotlib. The classifier
build additionally uses Biopython (installed at runtime; see `02_classifier/README.md`).
Scripts are written for a SLURM cluster (`sbatch <script>`); the embedded Python runs
inline within each `.slurm` wrapper.

> **Paths.** Scripts use absolute cluster paths (e.g. `~/sg/formal/merge4tubes/...`) and
> a hard-coded `sys.path` for `palettes_pub.py`. They reflect how the analysis was run;
> to re-execute elsewhere, adjust those paths (and the `palettes/` import) to your
> environment.

## Pipeline and run order

The stages run top to bottom. Within the taxonomic, alpha, and beta stages, the collapse/
compute step must precede its figure step.

### 1. Read processing → classification
*Located in `01_read_processing/`.*
Per tube: demultiplex, primer-trim (cutadapt), and denoise (DADA2). Tube 4 is the `-S4`
re-sequencing run.

| Script | Step |
|---|---|
| `tube1o8-July2nd.slurm` | tube 1 demux + trim + denoise |
| `tube2o8-demux-July9th.slurm`, `tube2o8-denoise-July9th.slurm` | tube 2 |
| `tube3o8-demux-July9th.slurm`, `tube3o8-denoise-July9th.slurm` | tube 3 |
| `tube4o8-demux-Oct.slurm`, `tube4o8-denoise-Oct.slurm` | tube 4 |
| `merge-4tubes-V4.slurm` | merge the four tables + rep-seqs, then `classify-sklearn` → `V4-merged-taxonomy.qza` |

DADA2 forward/reverse truncation lengths differ by run (forward 244–272, reverse
182–198). `merge-4tubes-V4.slurm` applies the classifier built in stage 2, so **stage 2
must be run before the merge step.**

### 2. Classifier build
*Located in `02_classifier/`.*
See `02_classifier/README.md`. Produces the naïve-Bayes SILVA 138.2 V3–V4
classifier consumed by `merge-4tubes-V4.slurm`.
Note: the classifier file is named `...-V4.qza` but is trained on the **V3–V4** region
(341F/805R) — the "V4" is a legacy label, not the region.

### 3. Filtering
*Located in `03_filtering/`.*

| Script | Step |
|---|---|
| `filter-remove-ch-mt.slurm` | remove Unassigned / mitochondria / chloroplast → `V4-merged-table-no-ch-filtered.qza` (the input table for all downstream stages) |

### 4. Taxonomic composition (Figures 1, 2, 5)
*Located in `04_taxonomic/`.*

| Script | Produces |
|---|---|
| `phylum-by-code.slurm` | phylum-level collapse (level 2) + relative frequency |
| `genus-collapse.slurm` | genus-level collapse (level 6) + relative frequency |
| `phylum-reorder-by-code-no57-v3q.slurm` | **Figure 1** — phylum stacked bars by body site × facility |
| `genus-heatmap-onepanel-v3p.slurm` | **Figure 2** — genus Log10 relative-abundance heatmap |
| `phylum_genus_2panel_v5_arial.slurm` | **Figure 5** — phylum + genus 2-panel (placenta / milk / neonate) |

The two collapse scripts must run before the figure scripts.

### 5. Alpha diversity (Figure 3)
*Located in `05_alpha/`.*

| Script | Produces |
|---|---|
| `alpha-rarefaction.slurm` | rarefaction curves (justifies the 10,000-read depth) |
| `alpha-by-species.slurm` | rarefies to 10,000 and computes observed features, Chao1, Shannon, Simpson, Pielou; splits by species (Tt / Sb) |
| `alpha-boxplots-Tt-no57-v3l.slurm` | **Figure 3** — Chao1 / Shannon / Pielou boxplots (*Tursiops*, dolphin 57 excluded) |
| `alpha-summary-by-bodysite-v2.slurm` | alpha statistics: Kruskal–Wallis + pairwise Mann–Whitney U, Benjamini–Hochberg FDR |

`alpha-by-species.slurm` must run before the boxplot and summary scripts.

### 6. Beta diversity (Figure 4)
*Located in `06_beta/`.*
Each panel script is self-contained: it filters the table to its subset, then runs
singleton removal → multiplicative zero replacement → CLR → Aitchison distance → PCoA,
and renders one panel.

| Script | Panel |
|---|---|
| `pcoa-2d-pub-V8d_arial_CLRaitchison_singleton-panelA.slurm` | Fig 4A — all body sites |
| `pcoa-genital-CLRaitchison.slurm` | Fig 4B — genital (vaginal + penile) |
| `pcoa-mammary-CLRaitchison.slurm` | Fig 4C — mammary |
| `pcoa-gastric-CLRaitchison.slurm` | Fig 4D — gastric |
| `pcoa-rectal-CLRaitchison.slurm` | Fig 4E — rectal |
| `pcoa-respiratory-CLRaitchison.slurm` | Fig 4F — respiratory |

The published Figure 4 arranges these six panels into a 2×3 grid with a shared legend;
the grid-assembly (cosmetic compositing) step is not included, so these scripts
reproduce the individual panels rather than the composited layout.

## Shared dependencies

- `metadata/merged_dolphin_4.txt` — sample metadata (defines SampleID, DolphinID, Species,
  BodySite, Facility/Code, Type, etc.); required by nearly every analysis script.
- `palettes/palettes_pub.py` — publication color palettes and body-site label normalization;
  imported by the taxonomic, alpha, and beta figure scripts.

## Notes and caveats

- **Fonts.** Figure scripts prefer Arial (expected under a local `fonts/` directory) and
  fall back to Liberation Sans / DejaVu Sans if it is absent. Arial `.ttf` files are
  proprietary and are not committed; figures render with the fallback fonts without
  Arial present.
- **`genus-collapse.slurm`** was reconstructed from the QIIME 2 provenance recorded in
  `genus-table.qza` (the original genus collapse was run interactively and not saved as a
  script); it reproduces the same command (`taxa collapse --p-level 6`).
- **Dolphin 57** is excluded from the analyses that report *Tursiops* results (the `no57`
  scripts), consistent across figures.
- **Not included:** the Figure 6 (Venn) analysis, which is based on an externally
  supplied table, is not part of this repository at this time.
