# SILVA 138.2 V3–V4 classifier build

This folder contains the code used to build the naïve-Bayes taxonomy classifier
applied in this study. Running these scripts in order reproduces the classifier
artifact (`silva-138-2-classifier-V4.qza`) from a fresh SILVA download.

> **Note on the filename.** The output classifier is named `...-V4.qza`, but it is
> trained on the **V3–V4** region (primers 341F / 805R; see step 5). The "V4" in the
> filename is a label carried over from an earlier naming choice, not a description of
> the region. The region is V3–V4.

## What you need first

Download the SILVA release **138.2** reference file from
[arb-silva.de](https://www.arb-silva.de/):

```
SILVA_138.2_SSURef_NR99_tax_silva.fasta.gz
```

(SSU Ref NR99, non-redundant at 99%.) SILVA ships this file gzipped and with RNA
sequences (U instead of T); the scripts below handle both facts.

### Decompress so that both forms are present

One script reads the **gzipped** file and another reads the **uncompressed** file, so
you need both present at once. Decompress with the `-k` (keep) flag rather than a plain
`gunzip`, which would delete the `.gz`:

```bash
gunzip -k SILVA_138.2_SSURef_NR99_tax_silva.fasta.gz
```

After this you have both `...tax_silva.fasta.gz` and `...tax_silva.fasta` in the
directory. This is the single most common thing to get wrong: a plain `gunzip`
consumes the `.gz`, and the taxonomy-extraction step then fails with "file not found."

## Run order

Each `.slurm` script submits to SLURM (`sbatch <script>`); the two `.py` files are
called by their matching `.slurm` wrappers, not run directly. All scripts assume the
SILVA files are in the working directory.

| # | Script | Reads | Produces |
|---|--------|-------|----------|
| 1 | `extract-silva-taxonomy.slurm` → `extract_silva_taxonomy.py` | `...tax_silva.fasta.gz` (gzipped) | `silva-138-2-taxonomy.txt` |
| 2 | `convert-rna-to-dna.slurm` → `convert_rna_to_dna.py` | `...tax_silva.fasta` (uncompressed) | `SILVA_138.2_SSURef_NR99_tax_silva_DNA.fasta` |
| 3 | `import-silva-seqs.slurm` | DNA FASTA from step 2 | `silva-138-2-seqs.qza` |
| 4 | `import-silva-taxonomy.slurm` | `silva-138-2-taxonomy.txt` from step 1 | `silva-138-2-tax.qza` |
| 5 | `extract-silva-v4.slurm` | `silva-138-2-seqs.qza` from step 3 | `silva-138-2-ref-seqs-V4.qza` |
| 6 | `train-silva-v4.slurm` | ref-seqs (step 5) + taxonomy (step 4) | `silva-138-2-classifier-V4.qza` |

Steps 1 and 2 are independent of each other (both read the raw SILVA file, in
different forms) and can run in either order once decompression is done. Steps 3–6 are
sequential. The primer-extraction step (5) uses:

- Forward 341F: `CCTACGGGNGGCWGCAG`
- Reverse 805R: `GACTACHVGGGTATCTAATCC`

(Klindworth et al. 2013, the standard Illumina V3–V4 pair.)

## Environment

- **QIIME 2 2024.2** (`module load qiime2/2024.2`) for steps 3–6.
- **Python 3 with Biopython** for steps 1–2. The scripts install Biopython at runtime
  (`pip install --user biopython`); no other Python packages are required. Biopython's
  `SeqIO` / `transcribe` API used here is stable across recent versions.

## Not included in this folder

These are regenerated by the scripts above or are external reference data, so they are
not committed here:

- The SILVA source files (`...tax_silva.fasta[.gz]`) — download from arb-silva.de.
- Large intermediate artifacts (`silva-138-2-seqs.qza`, `silva-138-2-tax.qza`,
  `silva-138-2-ref-seqs-V4.qza`) and the final `silva-138-2-classifier-V4.qza` —
  reproduced by running the pipeline.

This folder covers the **classifier build only**. Importing and denoising the study
reads, applying this classifier to them (`classify-sklearn`), and downstream diversity
and statistical analyses are documented elsewhere in the repository.
